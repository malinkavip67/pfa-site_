# PFA — рабочий журнал проекта

Последнее обновление: 15 июля 2026 года.

## О проекте

Premier Football Agency — многостраничный сайт футбольного агентства. Проект разрабатывается как поддерживаемый коммерческий продукт на Next.js 16 App Router, React, TypeScript Strict, Tailwind CSS и Framer Motion.

Локальный адрес: `http://localhost:3001/`

GitHub: `https://github.com/malinkavip67/pfa-site_`

Основная ветка: `main`

## Структура проекта

```text
app/                         Маршруты и общая сборка страниц
  contacts/                  Страница контактов
  news/                      Страница новостей
  partners/                  Страница партнёров
  players/                   Каталог и динамические страницы игроков
  layout.tsx                 Корневой layout
  page.tsx                   Главная страница
  not-found.tsx              Страница 404
  robots.ts                  Генерация robots.txt
  sitemap.ts                 Генерация sitemap.xml

components/
  layout/                    Header, Footer и общие элементы макета
  home/                      Секции главной страницы
  players/                   Компоненты игроков
  news/                      Компоненты новостей
  partners/                  Компоненты партнёров
  ui/                        Переиспользуемая дизайн-система

data/                        Типизированные статические данные
lib/                         Константы, Metadata и structured data
types/                       TypeScript-интерфейсы
styles/                      Глобальные стили и локальные шрифты
public/images/               Оптимизированные изображения проекта
```

Папка `hooks/` зарезервирована для действительно необходимых пользовательских хуков. Сейчас отдельных хуков проекту не требуется.

## Ключевые файлы

### Приложение

- `app/layout.tsx` — подключает локальные шрифты, Metadata, Header, Footer и Schema.org.
- `app/page.tsx` — собирает секции главной страницы и передаёт им данные.
- `styles/globals.css` — цвета бренда, базовая типографика, focus-visible и reduced-motion.
- `lib/constants.ts` — адрес сайта, контактный email и пункты навигации.
- `lib/metadata.ts` — единый генератор Metadata для страниц.
- `lib/structured-data.ts` — структурированные данные организации.

### Layout

- `components/layout/Header.tsx` — адаптивная шапка и доступное мобильное меню.
- `components/layout/Footer.tsx` — общий подвал сайта.
- `components/layout/StructuredData.tsx` — безопасный вывод JSON-LD.

### Главная страница

- `components/home/Hero.tsx` — главный экран с фоновым изображением, заголовком и CTA.
- `components/home/HeroStats.tsx` — статистика непосредственно под Hero.
- `components/home/About.tsx` — информация об агентстве и направления работы.
- `components/home/Services.tsx` — сетка услуг.
- `components/home/ServiceCard.tsx` — единая карточка услуги.
- `components/home/FeaturedPlayers.tsx` — избранные игроки.
- `components/home/Partners.tsx` — партнёры.
- `components/home/CTA.tsx` — финальный призыв к действию.

### Дизайн-система

- `components/ui/Container.tsx` — единая ширина и боковые отступы.
- `components/ui/Button.tsx` — общая кнопка и ссылка-кнопка.
- `components/ui/SectionHeading.tsx` — небольшие заголовки секций.
- `components/ui/Typography.tsx` — новая типографическая система; добавлена в текущей незавершённой задаче.
- `components/ui/AnimatedReveal.tsx` — переиспользуемая анимация появления.

### Данные и типы

- `data/players.ts` / `types/player.ts` — игроки.
- `data/news.ts` / `types/news.ts` — новости.
- `data/partners.ts` / `types/partner.ts` — партнёры.
- `data/services.ts` / `types/service.ts` — услуги и направления About.
- `data/stats.ts` / `types/stat.ts` — показатели главного экрана.

### Изображения

- `public/images/logo/logo-white.jpg` — утверждённый логотип PFA.
- `public/images/hero/hero-stadium.webp` — утверждённое изображение стадиона и футболиста.
- `public/images/players/` — локальные изображения карточек игроков.

## Постоянный порядок работы

### Обязательное правило дизайн-системы

- Любой новый компонент должен использовать существующие UI-компоненты `Typography`, `Button`, `SectionHeading`/`SectionTitle`, `Container` и `Card`.
- Если необходимого UI-компонента ещё нет, сначала создать переиспользуемый вариант в `components/ui`, затем использовать его в секциях и страницах.
- На всём сайте используется одна гарнитура — Manrope. Допустимы разные размеры, насыщенность и межбуквенные интервалы, но не другая семья шрифтов.
- Не создавать дубликаты существующих UI-компонентов.

После каждого этапа:

1. Внести изменения только локально и не переходить к следующему этапу автоматически.
2. Показать пользователю локальный результат и проверить затронутые компоненты на ширинах 1440, 1024, 768, 390 и 320 px.
3. Дождаться явного подтверждения дизайна.
4. Только после подтверждения запустить lint и build и исправить найденные ошибки.
5. Обновить `my-work-journal.md`.
6. Создать отдельный commit и отправить изменения на GitHub.
